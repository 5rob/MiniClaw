# Build Request: investigation-board

## Overview
Build Phase 1 of an interactive investigation board web app — a detective-style entity relationship visualiser. This is a standalone React app (NOT a MiniClaw skill) that should be created at the project root level in a folder called `investigation-board/`.

## What to Build

A fully interactive investigation board with seed data. No backend — everything runs client-side with hardcoded data for now.

### Core Components

1. **Board.tsx** — Main React Flow v12 canvas
   - Dark charcoal background (#1a1a2e) with React Flow's dot grid pattern
   - Pan via middle mouse / trackpad drag
   - Zoom via scroll wheel / pinch
   - Minimap in bottom-right corner
   - Zoom controls (zoom in/out/fit view buttons)
   - Keyboard: Escape to deselect, F to fit view

2. **EntityNode.tsx** — Custom React Flow node for primary entities
   - Circular photo area (120px) with fallback initials avatar (coloured circle with initials) when no photo
   - Name in bold below the photo
   - Entity type in smaller muted text below the name
   - Subtle drop shadow, glow effect on hover
   - Draggable (React Flow default)
   - When selected/clicked: subtle highlight ring

3. **MinorEntityNode.tsx** — Smaller node for non-primary entities
   - 60px circle, no photo, just initials or icon
   - Name label only
   - Thinner border, more muted appearance
   - Connected to primary entities via thin edges

4. **WordCloudCluster.tsx** — Per-entity floating word cloud
   - Render topic words as absolutely positioned HTML elements INSIDE a custom React Flow node (NOT as separate React Flow nodes — this is critical for performance)
   - Each word's font size is proportional to the count of data points for that entity+topic combination
   - Words are coloured by their topic colour
   - Use d3-force simulation PER ENTITY to make words gently float/orbit around their parent:
     - `forceRadial` centred on the entity to keep words in an orbit (radius ~150-200px)
     - `forceManyBody` with small negative charge so words repel each other (no overlap)
     - `forceCollide` based on word bounding box to prevent text overlap
     - Very low alpha decay (0.001) for continuous gentle floating animation
     - Low velocity decay (0.4) for smooth movement
   - Use requestAnimationFrame for smooth 60fps animation
   - Words should be interactive: hover shows count, click could filter sidebar

5. **ConnectionEdge.tsx** — Custom React Flow edge
   - Semi-transparent coloured lines matching topic colour
   - On hover: line brightens, tooltip shows connection summary + source title
   - On click: opens source_url in new tab (if available)
   - Topic name label on the edge (small, semi-transparent, visible on hover)
   - If multiple connections exist between the same two entities, bundle them visually (thicker line, tooltip lists all connections) — don't render 5 overlapping lines

6. **Sidebar.tsx** — Left sidebar panel
   - Fixed width (~320px), dark theme matching the board
   - Scrollable entity list with initials avatars and names
   - Click entity → board pans and zooms to centre on that entity (use React Flow's `fitBounds` or `setCenter`)
   - Search bar at top — filters entities by name
   - Topic filter section: checkboxes with coloured dots for each topic, toggle to show/hide topics on the board
   - Toggle switches: "Show Minor Entities", "Show Word Clouds"
   - When an entity is selected on the board, sidebar shows that entity's details: all data points grouped by topic, with source links

7. **App.tsx** — Main layout
   - Sidebar on the left, Board taking remaining space
   - Full viewport height, no scrolling on the body

        type: 'string',
        enum: ['action1', 'action2'],
        description: 'What operation to perform'
      }
      // ... more properties
    },
    required: ['action']
  }

  switch (action) {

      return { success: true, data: '...' };
    default:
      return { success: false, error: `Unknown action: ${action}` };
  }
}
