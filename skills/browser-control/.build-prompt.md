# Build Request: browser-control

## Overview
A comprehensive browser automation skill for MiniClaw based on OpenClaw's CDP (Chrome DevTools Protocol) architecture. This skill enables Claude to interact with websites programmatically through an isolated Chromium instance.

Core functionality:
1. **Element Snapshot System** - Scan web pages and assign numeric references to interactive elements so Claude can say "click [12]" instead of writing CSS selectors. Two modes: AI Snapshot (numbered list) and Role Snapshot (accessibility tree).

2. **Full Browser Control** - Navigate to URLs, wait for load states, click elements, type text, fill forms, scroll, hover, drag/drop, select dropdowns, take screenshots, export PDFs.

3. **Smart State Management** - Handle cookies, localStorage, sessionStorage, network headers, geolocation, timezone, device emulation (mobile/desktop viewports).

4. **Isolated Browser Profile** - Run in a separate Chromium instance (not Rob's personal browser) with headless support.

5. **Robust Wait Conditions** - Wait for URLs, elements, load states (load/domcontentloaded/networkidle), custom JavaScript conditions.

Technical requirements:
- Use Puppeteer (not Playwright) for CDP control - it's lighter and Node.js native
- Element snapshot system must inject numeric references into the DOM for easy identification
- All actions should return structured data (success/failure, element info, screenshots as base64 or file paths)
- Support both headless and headed modes (for debugging)
- Timeout protection (default 30s per action)
- Clean error handling with actionable messages

Actions needed:
- `navigate` - Open URL and wait for load state
- `snapshot` - Generate element snapshot (AI or Role mode)
- `click` - Click element by reference number or CSS selector
- `type` - Type text into input fields
- `fill_form` - Batch fill multiple form fields
- `screenshot` - Capture page or element screenshot
- `pdf` - Export page as PDF
- `get_cookies` - Retrieve cookies
- `set_cookies` - Set cookies
- `execute` - Run custom JavaScript in page context
- `wait` - Wait for conditions (URL, element, load state, custom)
- `close` - Close browser instance
- `status` - Check if browser is running

Use cases:
- Automating web form submissions
- Scraping content from dynamic websites
- Testing web applications
- Interacting with authenticated services
- Filling out online forms on Rob's behalf
- Taking screenshots of websites for reference

## Skill Architecture
This is a MiniClaw Discord bot skill. Each skill is a self-contained folder with:
- `handler.js` — The main module. MUST export:
  - `toolDefinition` — An Anthropic tool_use schema object ({ name, description, input_schema })
  - `execute(input)` — An async function that receives the tool input and returns a result object
- `SKILL.md` — Documentation: what the skill does, when to use it, example trigger phrases
- `PROGRESS.md` — Development log (append-only, timestamped entries)
- `data/` — Optional persistent data directory for JSON/text files

## Technical Requirements
- ES modules (import/export, no require())
- Node.js 20+ (no browser APIs)
- The skill runs inside a Discord bot on Windows — no network calls unless explicitly needed
- `process.cwd()` returns the MiniClaw project root
- The handler is loaded via dynamic import: `import('file:///path/to/handler.js')`
- All file operations should use `path.resolve()` for Windows compatibility
- Return objects from execute() — they get JSON-serialized and sent back to Claude API as tool results
- Errors should be returned as `{ success: false, error: 'message' }`, not thrown (unless truly fatal)

## Tool Definition Pattern

```javascript
export const toolDefinition = {

    type: 'object',

      action: {
        type: 'string',
        enum: ['action1', 'action2'],
        description: 'What operation to perform'
      }
      // ... more properties
    },
    required: ['action']
  }
};

export async function execute(input) {
  const { action } = input;
  switch (action) {
    case 'action1':
      return { success: true, data: '...' };
    default:
      return { success: false, error: `Unknown action: ${action}` };
  }
}
```
