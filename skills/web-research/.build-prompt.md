# Build Request: web-research

## Overview
A unified web research skill with three actions: search, fetch, and research. Uses Tavily AI as primary search engine with Brave Search API as fallback. Includes in-memory caching, API usage tracking, and comprehensive error handling. Zero npm dependencies - pure Node.js built-ins only.

ACTIONS:
1. search - Search the web using Tavily (primary) with Brave fallback
   - Tavily search params: query (required), search_depth ('basic' default, 'advanced' for complex), max_results (default 5), topic ('general'/'news'/'finance'), time_range, include_domains, exclude_domains, country (default 'australia')
   - ALWAYS include: include_answer: 'basic', include_raw_content: 'markdown', include_usage: true
   - Brave fallback when Tavily fails or returns errors (429, 432, 433, 500)
   - Return: Tavily's LLM answer, array of results with title/url/content/raw_content/score

2. fetch - Fetch and parse a specific URL
   - Uses Node.js https/http built-ins
   - Follow redirects (up to 5)
   - Parse HTML to clean readable text/markdown
   - Strip scripts, styles, nav elements
   - Max content length ~8000 chars (truncate with note)
   - Handle timeouts (10 second limit)

3. research - High-level orchestrator combining search + fetch
   - Searches via Tavily with include_raw_content enabled
   - Returns Tavily's LLM answer + top results with full content
   - If raw_content missing from results, optionally fetch top URLs
   - Comprehensive summary format

4. usage - Show API usage stats for current session
   - Track Tavily calls, Brave calls, cache hits
   - Show estimated credits used

CACHING:
- In-memory cache with 30-minute expiry
- Cache key = action + query/url hash
- Track cache hits in usage stats

TAVILY API:
- POST https://api.tavily.com/search
- Auth: Bearer token in Authorization header (from TAVILY_API_KEY env var)
- Request body is JSON with all parameters
- Response includes: query, answer, results[], images[], response_time, usage

BRAVE API:
- GET https://api.search.brave.com/res/v1/web/search?q={query}
- Auth: X-Subscription-Token header (from BRAVE_API_KEY env var)
- Response includes: web.results[] with title, url, description

ERROR HANDLING:
- Graceful fallback from Tavily to Brave
- Clear error messages for missing API keys
- Timeout handling for fetch
- Rate limit detection (429, 432, 433 from Tavily)

## Skill Architecture
This is a MiniClaw Discord bot skill. Each skill is a self-contained folder with:
- `handler.js` — The main module. MUST export:
  - `toolDefinition` — An Anthropic tool_use schema object ({ name, description, input_schema })
  - `execute(input)` — An async function that receives the tool input and returns a result object
- `SKILL.md` — Documentation: what the skill does, when to use it, example trigger phrases
- `PROGRESS.md` — Development log (append-only, timestamped entries)
- `data/` — Optional persistent data directory for JSON/text files

## Technical Requirements
- ES modules (import/export, no require())
- Node.js 20+ (no browser APIs)
- The skill runs inside a Discord bot on Windows — no network calls unless explicitly needed
- `process.cwd()` returns the MiniClaw project root
- The handler is loaded via dynamic import: `import('file:///path/to/handler.js')`
- All file operations should use `path.resolve()` for Windows compatibility
- Return objects from execute() — they get JSON-serialized and sent back to Claude API as tool results
- Errors should be returned as `{ success: false, error: 'message' }`, not thrown (unless truly fatal)
