# Integration Notes for MiniClaw Bot Developers

## Skill Ready Status: ⚠️ Partial (TTS/Whisper Ready, Voice Integration Required)

This skill is **structurally complete** but requires Discord voice integration to be fully functional.

## What Works Now

1. ✅ **TTS Synthesis** (`speak` action)
   - Creates OGG Opus audio files
   - Australian male voice (en-AU-Neural2-B)
   - Voice style modifiers (casual, professional, excited, calm)
   - Saves to `skills/voice-chat/data/`

2. ✅ **Setup Testing** (`test_setup` action)
   - Validates Whisper installation
   - Validates Google Cloud TTS credentials
   - Reports environment configuration
   - Checks Python dependencies

3. ✅ **Status Reporting** (`status` action)
   - Shows skill capabilities
   - Reports integration status
   - Lists available features

4. ✅ **Whisper Pipeline**
   - Supports faster-whisper (Python)
   - Supports whisper.cpp (C++)
   - Transcription logic complete
   - Just needs audio input from Discord

## What Needs Bot Integration

### Critical: @discordjs/voice Integration

The skill currently **cannot**:
- Join/leave voice channels
- Capture audio from voice channels
- Play TTS responses in voice channels

**Required Implementation in Main Bot:**

```javascript
// 1. Install packages
// npm install @discordjs/voice @discordjs/opus

// 2. Voice channel connection
import {
  joinVoiceChannel,
  createAudioPlayer,
  createAudioResource,
  EndBehaviorType
} from '@discordjs/voice';

// 3. Join voice channel (when user requests voice chat)
const connection = joinVoiceChannel({
  channelId: channel.id,
  guildId: guild.id,
  adapterCreator: guild.voiceAdapterCreator,
});

// 4. Capture audio from user
const receiver = connection.receiver;
const audioStream = receiver.subscribe(userId, {
  end: {
    behavior: EndBehaviorType.AfterSilence,
    duration: 500, // ms of silence
  },
});

// 5. Save audio stream to file
const writeStream = fs.createWriteStream('./temp-audio.pcm');
audioStream.pipe(writeStream);

// 6. When recording done, call skill's transcription
// (The skill's handler.js has transcribeAudio() function ready)

// 7. Play TTS response
const player = createAudioPlayer();
const resource = createAudioResource('/path/to/tts-file.opus');
player.play(resource);
connection.subscribe(player);
```

## How Bot Should Use This Skill

### Scenario 1: User Requests Voice Response

```
User (in voice channel): "Say hello in voice"

Bot flow:
1. Claude API decides to use voice_chat tool
2. Calls: voice_chat({ action: 'speak', text: 'Hello mate!', voice_style: 'casual' })
3. Skill returns: { success: true, audio_file: '/path/to/tts.opus' }
4. Bot plays audio_file in voice channel (using @discordjs/voice)
```

### Scenario 2: User Wants Voice Conversation

```
User (in voice channel): "Can we talk?"

Bot flow:
1. Bot joins voice channel (using @discordjs/voice)
2. Bot starts audio capture
3. User speaks
4. Bot saves audio to temp file
5. Bot calls skill's transcribeAudio() OR sends audio to external service
6. Transcription sent to Claude API
7. Claude processes with full SOUL/IDENTITY context
8. Claude uses voice_chat({ action: 'speak', text: response })
9. Bot plays TTS in voice channel
```

### Scenario 3: Test Setup

```
User: "Test voice chat setup"

Bot flow:
1. Claude API uses voice_chat({ action: 'test_setup' })
2. Skill checks Whisper, TTS, environment
3. Returns diagnostic info
4. Bot displays results to user
```

## File Paths and Windows Compatibility

All file operations use `path.resolve()` for Windows compatibility:

```javascript
const dataDir = path.resolve(__dirname, 'data');
const logsDir = path.resolve(__dirname, 'logs');
const outputPath = path.resolve(dataDir, `tts-${timestamp}.opus`);
```

Python scripts use escaped backslashes:
```javascript
audio_path = "${audioFilePath.replace(/\\/g, '\\\\')}"
```

## Error Handling

The skill uses **non-throwing errors** as per MiniClaw conventions:

```javascript
return { success: false, error: 'detailed error message' };
```

All errors logged to `logs/voice-chat-YYYY-MM-DD.log` with:
- Timestamp
- Log level (INFO, ERROR)
- Message
- Metadata (JSON)

## Performance Expectations

- **Whisper (GPU)**: 1-2 seconds for 10-second audio
- **TTS synthesis**: 0.5-1 second
- **Total latency goal**: 2-4 seconds (capture → response)

## Future Delegation Architecture

The skill is designed for future task routing:

```javascript
// Pipeline: captureAudio() → transcribe() → process(model, immediate) → speak()

// Simple question (immediate voice response)
process(model: 'haiku', immediate: true) → speak()

// Complex request (background processing)
process(model: 'sonnet', immediate: false) → notify when done
```

This allows:
- Haiku handles quick voice interactions
- Sonnet/Opus processes complex tasks in background
- User gets immediate acknowledgment: "On it, I'll let you know when done"
