# Build Request: voice-chat

## Overview
Complete the voice conversation loop for the voice-chat skill. Currently the bot can join/leave voice channels and has working TTS + Whisper transcription in isolation, but needs the orchestration layer that actively listens to the user's voice in the channel, transcribes it, gets a Claude (Haiku) response, generates TTS, and plays it back.

This build needs to:
1. Start audio capture when the bot joins the voice channel
2. Detect voice activity (when user starts/stops speaking)
3. Buffer the audio while user is speaking
4. When user stops speaking, save buffer → transcribe with Whisper
5. Send transcription to Claude Haiku for a response
6. Generate TTS from response
7. Play TTS back through the voice channel
8. Return to listening state

The infrastructure is already built:
- voice-manager.js handles join/leave
- audio-capture.js has the capture pipeline (opus → PCM)
- audio-playback.js handles TTS playback
- handler.js has transcribeAudio() and synthesizeSpeech() working

What's needed is the event-driven loop that wires these together and manages the conversation state (idle → listening → processing → speaking → back to listening).

Consider using Discord.js voice's speaking events to detect when the user starts/stops talking. The bot should be resilient to errors (don't crash if transcription fails, just log and return to listening).

## Skill Architecture
This is a MiniClaw Discord bot skill. Each skill is a self-contained folder with:
- `handler.js` — The main module. MUST export:
  - `toolDefinition` — An Anthropic tool_use schema object ({ name, description, input_schema })
  - `execute(input)` — An async function that receives the tool input and returns a result object
- `SKILL.md` — Documentation: what the skill does, when to use it, example trigger phrases
- `PROGRESS.md` — Development log (append-only, timestamped entries)
- `data/` — Optional persistent data directory for JSON/text files

## Technical Requirements
- ES modules (import/export, no require())
- Node.js 20+ (no browser APIs)
- The skill runs inside a Discord bot on Windows — no network calls unless explicitly needed
- `process.cwd()` returns the MiniClaw project root
- The handler is loaded via dynamic import: `import('file:///path/to/handler.js')`
- All file operations should use `path.resolve()` for Windows compatibility
- Return objects from execute() — they get JSON-serialized and sent back to Claude API as tool results
- Errors should be returned as `{ success: false, error: 'message' }`, not thrown (unless truly fatal)
